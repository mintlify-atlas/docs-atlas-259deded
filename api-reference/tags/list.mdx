---
title: "List tags"
description: "Get all tags for the authenticated user"
---

<Info>
  All requests to this endpoint require an `Authorization: Bearer <token>` header.
</Info>

## Request

**GET** `/api/tags`

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token. Example: `Bearer eyJhbGci...`
</ParamField>

## Response

Returns an array of tag objects belonging to the authenticated user.

<ResponseField name="[]" type="object[]">
  <Expandable title="Tag object" defaultOpen>
    <ResponseField name="id" type="string" required>
      Unique identifier for the tag (UUID).
    </ResponseField>
    <ResponseField name="name" type="string" required>
      Display name of the tag. Unique per user.
    </ResponseField>
    <ResponseField name="color" type="string">
      Optional hex color string associated with the tag. `null` if not set.
    </ResponseField>
    <ResponseField name="isDeleted" type="boolean" required>
      Whether the tag has been soft-deleted. Tags with `isDeleted: true` are included in the response for offline sync purposes.
    </ResponseField>
    <ResponseField name="userId" type="string" required>
      ID of the user who owns this tag.
    </ResponseField>
    <ResponseField name="createdAt" type="string" required>
      ISO 8601 timestamp of when the tag was created.
    </ResponseField>
    <ResponseField name="updatedAt" type="string" required>
      ISO 8601 timestamp of when the tag was last updated.
    </ResponseField>
  </Expandable>
</ResponseField>

<Note>
  Tags with `isDeleted: true` are returned alongside active tags. This is intentional for offline-first sync — clients should use `isDeleted` to reconcile local state rather than treating a missing tag as never having existed.
</Note>

<CodeGroup>
